
# Reportify Plus

**Reportify Plus** — микросервисная версия проекта [Reportify](https://github.com/ummuys/reportify) для асинхронной генерации отчётов из SQL-запросов.

Клиент создаёт задачу на генерацию отчёта, backend сохраняет метаданные, отправляет задачу в Kafka, worker обрабатывает её, формирует файл и сохраняет результат в MinIO. Готовый отчёт можно получить по ссылке.

> Статус: проект находится в активной разработке.

---

## Возможности

- создание задач на генерацию отчётов;
- отслеживание статуса генерации;
- поддержка форматов: PDF, XLSX, CSV, JSON, DOCX;
- хранение файлов в MinIO;
- выдача файлов через presigned-ссылки;
- кэширование статусов в Redis;
- JWT-аутентификация;
- пользователи и роли;
- REST API через gateway;
- Swagger-документация;
- внутреннее взаимодействие сервисов через gRPC;
- асинхронная обработка задач через Kafka;
- DLQ для ошибочных задач;
- просмотр схем, таблиц и колонок источника данных.

## Сервисы

### `gateway`

Внешний HTTP API.

Отвечает за:

* обработку REST-запросов;
* авторизацию пользователей;
* проверку JWT;
* проксирование запросов во внутренние gRPC-сервисы;
* Swagger-документацию.

### `auth`

Сервис пользователей и авторизации.

Отвечает за:

* регистрацию и вход;
* работу с пользователями;
* управление ролями;
* генерацию и проверку JWT.

### `report`

Сервис отчётов.

Отвечает за:

* создание задач на генерацию отчётов;
* хранение метаданных отчётов;
* получение статусов;
* работу со схемами, таблицами и колонками источника данных;
* публикацию задач в Kafka.

### `report-worker`

Фоновый обработчик задач.

Отвечает за:

* чтение задач из Kafka;
* выполнение SQL-запросов;
* генерацию файлов отчётов;
* загрузку файлов в MinIO;
* обновление статуса задачи;
* обработку ошибочных задач через DLQ.

---

## Как работает генерация отчёта

1. Клиент отправляет запрос на создание отчёта в `gateway`.
2. `gateway` проверяет JWT и передаёт запрос в `report`.
3. `report` сохраняет метаданные задачи в PostgreSQL.
4. `report` публикует задачу в Kafka.
5. `report-worker` забирает задачу из Kafka.
6. Worker выполняет SQL-запрос и формирует файл отчёта.
7. Файл сохраняется в MinIO.
8. Статус задачи обновляется и кэшируется в Redis.
9. Клиент получает ссылку на готовый файл.

## Быстрый старт

### 1. Клонировать репозиторий

```bash
git clone https://github.com/ummuys/reportify_plus.git
cd reportify_plus
```

### 2. Создать `.env`

```bash
cp .env.example .env
```

Заполните значения переменных окружения в `.env`.

### 3. Запустить проект через Docker

```bash
cd deployment
make up
```

После запуска приложение будет доступно через Nginx:

```text
http://localhost:8088
```
